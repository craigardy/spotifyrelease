About the Application

Stay up to date with new music from your favorite artists on Spotify. This app sends you email updates about the latest releases from the artists you follow.

When you sign up with your email, you'll be asked to authorize access to your followed artists on Spotify. Once authorized, you'll receive an initial email with any new music they've released in the past 4 weeks.

After that, you'll get updates every two weeks so you never miss a new release.

We request permission to view the artists you follow and the music in your Spotify library. This allows us to identify recent releases from your favorite artists and ensure you don't miss anything you've saved or followed. Your email is used solely to send updates — nothing else.

Developer Info

This project demonstrates my skills in several key areas:

  • Backend:
    • Flask: Used for building the web application's backend, handling routing, managing user sessions, and integrating with the Spotify API.
    • Flask Blueprints: Organized application routes using Flask Blueprints for better code modularity and maintainability.
    • SQLAlchemy: Employed as an ORM to interact with the SQLite database, defining the data models and performing database operations.
    • Email Handling: Automated email notifications using Python's email.mime library within the Flask application.
    • Flask's `redirect`, `url_for`, and `session`: Implemented user authentication and navigation using Flask's features for redirects and session management.
  • API Interaction:
    • Spotipy: Utilized the Spotipy library to interact with the Spotify Web API, retrieving artist and release information.
    • `concurrent.futures.ThreadPoolExecutor`: Utilized to efficiently fetch data from the Spotify API by making concurrent requests.
  • Frontend:
    • JavaScript: Implemented client-side email validation to enhance user experience and ensure data integrity.
    • Regular Expressions (`re`): Employed for robust validation of user-provided email addresses.
  • Database:
    • SQLite: Used for local data storage.
  • Configuration Management:
    • Flask Configuration Classes: Utilized to handle different application environments (development, production), allowing for customized settings.
    • `python-dotenv`: Used to manage environment variables, ensuring secure storage of sensitive information like API keys.
  • Deployment:
    • `waitress`: (or Gunicorn/uWSGI) For production deployments, the application is served using `waitress`, a production-ready WSGI server.
  • Version Control:
    • Git: Used for version control and collaboration, with the source code available on GitHub.
